TurtleTrack — Setup Guide
=========================

This guide walks through setting up TurtleTrack for development and first use.


PART 1: ENVIRONMENT SETUP
--------------------------

1. Copy the environment file:
   cp .env.example .env

2. Key variables to configure:

   NODE_ENV=development
   PORT=3000
   MONGO_URI=mongodb://localhost:27017/turtletrack
   ML_SERVICE_URL=http://localhost:8000
   UPLOAD_DIR=../uploads

   For physical device testing, replace localhost with your computer's IP:
   ML_SERVICE_URL=http://192.168.1.x:8000


PART 2: DATABASE SETUP
-----------------------

Option A — Docker (Recommended):
  docker-compose up mongo -d

Option B — Local MongoDB:
  Ensure MongoDB 7.0+ is running on localhost:27017
  The database "turtletrack" is created automatically on first connection.


PART 3: BACKEND NODE SETUP
---------------------------

  cd backend-node
  npm install
  npm run dev

  Verify: curl http://localhost:3000/api/health


PART 4: PYTHON ML SERVICE SETUP
---------------------------------

  cd backend-ml

  # Create virtualenv
  python -m venv .venv

  # Activate (Windows PowerShell)
  .venv\Scripts\Activate.ps1

  # Activate (Windows CMD)
  .venv\Scripts\activate.bat

  # Activate (macOS/Linux)
  source .venv/bin/activate

  # Install dependencies
  pip install -r requirements.txt

  # Start service
  python main.py

  Notes:
  - First startup downloads MobileNetV2 weights from PyTorch Hub (~14MB)
  - FAISS index is created fresh if no saved index exists
  - Interactive docs: http://localhost:8000/docs

  Verify: curl http://localhost:8000/ml/health


PART 5: FRONTEND SETUP
-----------------------

  cd frontend
  npm install
  npx expo start

  iOS:
    - Install "Expo Go" from App Store
    - Scan the QR code shown in terminal

  Android:
    - Install "Expo Go" from Play Store
    - Scan the QR code shown in terminal

  Physical device note:
    Update frontend/src/constants/theme.ts:
      API_BASE_URL = 'http://YOUR_IP:3000/api'
      UPLOADS_BASE_URL = 'http://YOUR_IP:3000'


PART 6: FIRST USE WORKFLOW
---------------------------

1. Open TurtleTrack on your phone
2. Go to "Identify" tab
3. Take a photo of a sea turtle (focus on neck region)
4. Tap "Run Identification"
5. If no match: submitted as pending verification
6. Go to "Verify" tab → Tap the pending item → Approve
7. New turtle profile is created and ML index updated
8. Next time the same turtle is photographed → will match!


PART 7: ADDING TURTLES MANUALLY
---------------------------------

Via API (curl example):

  curl -X POST http://localhost:3000/api/turtles \
    -F "species=green" \
    -F "gender=female" \
    -F "firstSightingDate=2014-06-15" \
    -F "birthLocation=Great Barrier Reef, Australia" \
    -F "notes=First sighting near Heron Island" \
    -F "profileImage=@/path/to/turtle.jpg"

The turtle will receive an auto-generated ID like TT-A1B2C3D4.

To register the turtle's image in the ML index:
  POST http://localhost:8000/ml/register
  (multipart: turtle_id=TT-X, file=<image>)


PART 8: DOCKER FULL STACK
--------------------------

  docker-compose up --build

  This starts:
    - MongoDB on port 27017
    - Node API on port 3000
    - ML Service on port 8000

  Stop everything:
    docker-compose down

  Reset data:
    docker-compose down -v  (removes volumes including database)


PART 9: DEVELOPMENT TIPS
--------------------------

- Use http://localhost:8000/docs for interactive ML API testing
- Monitor MongoDB with MongoDB Compass connecting to localhost:27017
- The FAISS index is saved to backend-ml/storage/index/ — do not delete
- Uploaded images are in uploads/ — symlinked in Docker volumes
- Node backend hot-reloads with ts-node-dev
- Python ML service hot-reloads with uvicorn --reload (set in main.py)
